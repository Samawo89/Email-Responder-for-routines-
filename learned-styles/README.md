# learned-styles/

The routine's memory. One markdown file per person you correspond with, named by a slug of
their email (`jane.smith@club.com` → `jane-smith-club-com.md`).

Each file captures **how you reply to that specific person** — greeting, sign-off, tone,
length, recurring context — learned from your past Sent emails. The routine:

1. **Reads** the file before drafting a reply to that person.
2. **Updates** it after each run, sharpening the profile over time.

This is what makes the responder "learn over time" — and because it lives in the repo, the
knowledge persists across runs and is available when the routine runs remotely.

See [`_TEMPLATE.md`](_TEMPLATE.md) for the file shape. Profiles are created automatically;
you can also hand-edit any file to correct the routine.
